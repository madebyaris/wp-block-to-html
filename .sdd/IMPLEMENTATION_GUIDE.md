# Implementation Guide

Use this guide with `/implement` and `specs/active/[task-id]/todo-list.md`.

## Execution Rules
1. Read `plan.md`, `spec.md`, `tasks.md`, and `todo-list.md` before starting.
2. Execute in dependency order.
3. Mark each completed item immediately.
4. Document blockers instead of skipping them.
5. Verify code, tests, and follow-up notes before declaring completion.

## File Locations
- Active work: `specs/active/[task-id]/`
- Completed work: `specs/completed/[task-id]/`
- Historical migrated context: `specs/completed/legacy-memory-bank/` and other completed spec folders
